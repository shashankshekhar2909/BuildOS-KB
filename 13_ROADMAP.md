# 13 — Roadmap

## Phase 1 — Backend Foundation (Week 1)

**Goal:** API running, projects discovered, stored in DB.

### Tasks
- [ ] Project scaffold: `uv init`, FastAPI app factory, config, lifespan
- [ ] PostgreSQL setup: Docker Compose, pgvector extension, schemas
- [ ] Redis setup: Docker Compose, ARQ worker skeleton
- [ ] Alembic migrations: all tables from `03_DATABASE_SCHEMA.md`
- [ ] `DiscoveryService`: scan directories, detect framework, upsert projects
- [ ] `GET /api/projects` — list all projects
- [ ] `GET /api/projects/{slug}` — get single project
- [ ] `POST /api/admin/index/full` — trigger discovery job
- [ ] `GET /api/admin/health` — DB + Redis health check
- [ ] ARQ discovery cron (every 15 min)

**Done when:** `curl localhost:8000/api/projects` returns projects from `~/project`.

---

## Phase 2 — Context Extraction (Week 2)

**Goal:** Documents extracted and stored. Stack detected.

### Tasks
- [ ] `ExtractionService`: read priority files, hash change detection
- [ ] Document parsers: `package.json`, `pyproject.toml`, `docker-compose.yml`
- [ ] `GET /api/documents` and `GET /api/projects/{slug}/documents`
- [ ] `project_technologies` populated from extracted manifests
- [ ] Technology normalization map
- [ ] `GET /api/projects` returns `technologies[]` array
- [ ] ARQ extraction job wired to discovery event

**Done when:** CLAUDE.md, README.md, package.json parsed and visible in API for all projects.

---

## Phase 3 — OKF Generation (Week 3)

**Goal:** Every project has a `buildos.okf.md` generated by AI.

### Tasks
- [ ] LiteLLM setup: config, model routing, retry logic
- [ ] `OKFService`: prompt builder, LLM call, post-processing, disk write
- [ ] `DocumentChunk` table and `EmbeddingService`: chunking with tiktoken
- [ ] Embedding pipeline: batch embed documents
- [ ] `tsvector` update after chunk insert
- [ ] `GET /api/projects/{slug}/okf` route
- [ ] ARQ OKF generation job wired to extraction completion
- [ ] OKF override detection (don't overwrite user-written OKFs)

**Done when:** `buildos.okf.md` exists in all project directories and matches expected schema.

---

## Phase 4 — Search (Week 4)

**Goal:** Hybrid search working end-to-end.

### Tasks
- [ ] `SearchService`: keyword, semantic, graph, merge
- [ ] `GET /api/search` with all three modes
- [ ] Search filters: language, framework, document_type, project_slug
- [ ] Redis cache (5-min TTL, pattern invalidation on re-index)
- [ ] `GET /api/search/keyword`, `/semantic`, `/graph` endpoints
- [ ] Search history logging

**Done when:** `curl "localhost:8000/api/search?q=FastAPI+Redis"` returns relevant chunks from multiple projects.

---

## Phase 5 — MCP Server (Week 5)

**Goal:** Claude Code can query all knowledge via MCP tools.

### Tasks
- [ ] MCP server scaffold (`fastmcp` or `mcp` SDK)
- [ ] All 6 tools implemented: `list_projects`, `get_project`, `search`, `related`, `get_okf`, `reindex`
- [ ] MCP server runnable via stdio: `uv run python -m app.mcp_server`
- [ ] MCP server runnable via HTTP/SSE on port 8100
- [ ] Rate limiting on reindex tool
- [ ] Input validation on all tools
- [ ] Claude Code config: `claude mcp add buildos-kb ...`
- [ ] Manual test: Claude answers "what projects use FastAPI?" correctly

**Done when:** Claude Code session uses `search` tool and returns correct project context without manual prompting.

---

## Phase 6 — Frontend (Week 6)

**Goal:** Web UI for browsing and searching knowledge.

### Tasks
- [ ] Next.js 15 scaffold: app router, Carbon theme, React Query setup
- [ ] `AppShell`: Carbon SideNav + Header
- [ ] Dashboard page: stats + system health
- [ ] Projects list page: table with filters
- [ ] Project detail: tabbed (Overview, Architecture, Knowledge, Files)
- [ ] Search page: universal search box, results, filters
- [ ] Graph page: React Flow with project nodes + relationship edges
- [ ] Project detail → Graph tab (subgraph view)
- [ ] "Re-index" button on project detail
- [ ] Dark theme default (Carbon `g100`)

**Done when:** Full UI working for all 6 pages. Search returns results in < 500ms.
